คุณสมบัติหลัก

ทุกสิ่งที่ต้องการสำหรับการพัฒนา API ระดับ Production

🔐
JWT Authentication
ระบบยืนยันตัวตนด้วย JSON Web Token พร้อม refresh token rotation และ revoke list เพื่อความปลอดภัยสูงสุด
🛡️
CSRF Protection
ป้องกัน Cross-Site Request Forgery ทุก POST endpoint ต้องส่ง X-CSRF-TOKEN header ที่ได้จาก /auth/csrf-token
⚡
Router อัตโนมัติ
ระบบ routing อัตโนมัติผ่านรูปแบบ /api/{module}/{method}/{action} ไม่ต้องกำหนด route ทีละเส้น
🗄️
Multi-Database
รองรับ MySQL, PostgreSQL, MSSQL และ SQLite ผ่าน Query Builder ที่ใช้งานง่าย ป้องกัน SQL Injection ในตัว
📋
Standard Response
ทุก endpoint คืนค่า JSON Envelope มาตรฐาน: success, code, message, data, errors, request_id
🚦
Rate Limiting
จำกัดจำนวน request ต่อ IP เพื่อป้องกัน brute force และ DDoS พร้อมกำหนดค่าได้ตามต้องการ
❤️
Health Check
Endpoint /system/health, /system/readiness และ /system/version สำหรับ monitoring และ DevOps pipeline
📄
OpenAPI 3.0
สเปกครบถ้วนในไฟล์ openapi.yaml สามารถ generate SDK อัตโนมัติสำหรับ TypeScript, Python และอื่นๆ
🔔
Multi-Channel Notify
ส่งการแจ้งเตือนผ่าน Email, Telegram, LINE, SMS รองรับเทมเพลตอีเมล HTML ในตัว

API Endpoints

Endpoints ทั้งหมดที่พร้อมใช้งาน ทดสอบแล้วบน localhost

Method Endpoint หน้าที่ Auth CSRF
GET /api/index/auth/csrf-token ออก CSRF token สำหรับ POST request - -
GET /api/index/auth/settings อ่าน feature flags (register, forgot, activate) - -
GET /api/index/auth/verify ตรวจสอบ access token ปัจจุบัน ✅ -
GET /api/index/auth/me ดึงข้อมูลผู้ใช้ปัจจุบัน ✅ -
GET /api/index/auth/activate?id={code} ยืนยันบัญชีด้วย activation code - -
POST /api/index/auth/login เข้าสู่ระบบ → คืน access + refresh token - ✅
POST /api/index/auth/logout ออกจากระบบ revoke token ปัจจุบัน - ✅
POST /api/index/auth/refresh ต่ออายุ access token ด้วย refresh token - ✅
POST /api/index/auth/register สมัครสมาชิกใหม่ - ✅
POST /api/index/auth/update อัปเดตโปรไฟล์ผู้ใช้ปัจจุบัน ✅ ✅
POST /api/index/auth/forgot ขอรีเซ็ตรหัสผ่าน → ส่งอีเมล - ✅
POST /api/index/auth/resetpassword รีเซ็ตรหัสผ่านด้วย token จากอีเมล - ✅
GET /api/index/system/health Liveness probe — ระบบยังทำงานอยู่ - -
GET /api/index/system/readiness Readiness probe — config/storage/database - -
GET /api/index/system/version เวอร์ชันระบบปัจจุบัน - -
GET /api/index/system/time เวลาเซิร์ฟเวอร์ (timezone: Asia/Bangkok) - -
GET /api/index/system/features Feature flags ทั้งหมดของระบบ - -

✅ Auth = ต้องใส่ Bearer token ใน Authorization header  |  ✅ CSRF = ต้องส่ง X-CSRF-TOKEN header

ตัวอย่างการทดสอบ API

คำสั่ง curl พร้อมใช้งาน คัดลอกได้ทันที

🔑 ขอ CSRF Token GET 
# เก็บ cookie และดึง CSRF token จาก header
curl -s -c /tmp/k.cookie -D /tmp/k.headers \
  http://localhost/kotchasan-api/api/index/auth/csrf-token

# อ่าน token จาก response header
TOK=$(grep -i '^X-Csrf-Token:' /tmp/k.headers \
  | awk '{print $2}' | tr -d '\r')
echo "CSRF Token: $TOK"
🔐 Login POST 
curl -s -b /tmp/k.cookie -c /tmp/k.cookie \
  -H "X-CSRF-TOKEN: $TOK" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"password123"}' \
  http://localhost/kotchasan-api/api/index/auth/login

# เก็บ access token สำหรับคำสั่งถัดไป
ACCESS=$(curl ... | python3 -c \
  "import sys,json; print(json.load(sys.stdin)['data']['access_token'])")
👤 ดึงข้อมูลโปรไฟล์ (me) GET 
curl -s -b /tmp/k.cookie \
  -H "Authorization: Bearer $ACCESS" \
  http://localhost/kotchasan-api/api/index/auth/me
🔄 Refresh Token POST 
curl -s -b /tmp/k.cookie -c /tmp/k.cookie \
  -H "X-CSRF-TOKEN: $TOK" \
  -H "Content-Type: application/json" \
  -d '{"refresh_token":"eyJhbGciOiJIUzI1..."}' \
  http://localhost/kotchasan-api/api/index/auth/refresh
📝 สมัครสมาชิก POST 
curl -s -b /tmp/k.cookie -c /tmp/k.cookie \

Database Schema

SQL ที่ต้องสร้างก่อนใช้งาน API — รองรับ MySQL / MariaDB

📋 ตาราง user ใช้โดย: auth/login, auth/me, auth/register, auth/update, auth/forgot, auth/resetpassword, auth/verify, auth/activate 
CREATE TABLE `user` (
  `id`           int(11)         NOT NULL AUTO_INCREMENT,
  `username`     varchar(100)   NOT NULL DEFAULT ''   -- ใช้ login (รองรับ email),
  `password`     varchar(64)    NOT NULL DEFAULT ''   -- SHA1 hash,
  `salt`         varchar(32)    NOT NULL DEFAULT ''   -- random salt สำหรับ hash,
  `name`         varchar(200)   NOT NULL DEFAULT ''   -- ชื่อแสดง,
  `phone`        varchar(20)    DEFAULT NULL,
  `id_card`      varchar(20)    DEFAULT NULL             -- เลขบัตรประชาชน (optional),
  `telegram_id`  varchar(100)   DEFAULT NULL,
  `status`       tinyint(1)    NOT NULL DEFAULT 0    -- 0=user, 1=admin,
  `social`       varchar(20)    NOT NULL DEFAULT 'user' -- 'user' | 'google' | 'facebook',
  `active`       tinyint(1)    NOT NULL DEFAULT 1    -- 1=active, 0=รอยืนยันอีเมล,
  `activatecode` varchar(64)    NOT NULL DEFAULT ''   -- email activation / password reset token,
  `permission`   text           DEFAULT NULL             -- JSON permissions,
  `token`        varchar(512)   DEFAULT NULL             -- JWT access token ปัจจุบัน,
  `token_expires` datetime      DEFAULT NULL             -- วันหมดอายุ access / reset token,
  `visited`      int(11)        NOT NULL DEFAULT 0    -- จำนวนครั้ง login,
  `created_at`   datetime       DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `username` (`username`),
  KEY `active` (`active`),
  KEY `status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
📋 ตาราง logs ใช้โดย: auth/login, auth/logout, auth/register, auth/forgot, auth/resetpassword 
CREATE TABLE `logs` (
  `id`         int(11)         NOT NULL AUTO_INCREMENT,
  `src_id`     int(11)         NOT NULL DEFAULT 0    -- ID ของ resource ที่ถูกกระทำ (เช่น user.id),
  `member_id`  int(11)         NOT NULL DEFAULT 0    -- ID ผู้กระทำ (0 = guest),
  `module`     varchar(50)    NOT NULL DEFAULT ''   -- เช่น 'index',
  `action`     varchar(50)    NOT NULL DEFAULT ''   -- เช่น 'login', 'register', 'logout',
  `topic`      varchar(255)   NOT NULL DEFAULT ''   -- ข้อความสรุป,
  `reason`     varchar(255)   DEFAULT NULL             -- เหตุผล (เช่น กรณี reject),
  `datas`      text           DEFAULT NULL             -- JSON ข้อมูลเพิ่มเติม,
  `created_at` datetime       DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `member_id` (`member_id`),
  KEY `module_action` (`module`, `action`),
  KEY `created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
🔗 Endpoint → Column ที่ใช้งาน
Endpoint READ columns WRITE columns
POST /auth/login username, password, salt, active, activatecode, status, permission token, token_expires, visited · logs: action=login
GET /auth/me id, username, name, phone, status, permission, token, token_expires —
GET /auth/verify id, token, token_expires, active —
POST /auth/register username (duplicate check) INSERT ทุก column · logs: action=register
POST /auth/update id, token (auth check) name, phone, password, salt
POST /auth/logout — token=NULL, token_expires=NULL · logs: action=logout
POST /auth/refresh id, active (via JWT payload) token, token_expires · revoked_tokens.json
GET /auth/activate activatecode (WHERE) active=1, activatecode=''
POST /auth/forgot username (find user) activatecode (hash), token_expires · logs: action=forgot
POST /auth/resetpassword id, activatecode, token_expires password, salt, activatecode='', token_expires=NULL · logs: action=reset

ไฟล์ cache ที่ใช้เสริม: datas/cache/auth_sessions.json (active sessions) และ datas/cache/revoked_tokens.json (revoked refresh tokens) — ต้อง writable โดย web server

ความปลอดภัย

4 ชั้นการป้องกันตาม OWASP Best Practices

🔑
JWT + Refresh Token Rotation
Access token หมดอายุสั้น, Refresh token ถูก rotate ทุกครั้งที่ใช้ และ revoke list ป้องกันการใช้ token เก่า
🛡️
CSRF Double Submit Cookie
ทุก POST ต้องมี X-CSRF-TOKEN ที่ตรงกับ session cookie ป้องกัน Cross-Site Request Forgery อย่างสมบูรณ์
🚦
Rate Limiting per IP
จำกัดจำนวน request ต่อ IP ต่อนาที กำหนดค่าได้ใน config ป้องกัน brute force และ automated attacks
🌐
CORS Origin Allowlist
จำกัด origin ที่อนุญาตผ่าน api_cors config ป้องกัน unauthorized cross-origin requests
🔒
IP Allowlist
กำหนด api_ips ใน config เพื่อจำกัดว่า IP ใดสามารถเรียก API ได้ ใช้ 0.0.0.0 เพื่อเปิดทุก IP
💉
SQL Injection Prevention
Query Builder ใช้ Prepared Statement ทุกครั้ง ไม่มีการต่อ string SQL โดยตรง ปลอดภัยจาก SQL Injection
🔐
Password Hashing
รหัสผ่านถูก hash ด้วย bcrypt/Argon2 ผ่าน password_hash() ไม่มีการเก็บ plain text
📋
Input Validation
Validator class ตรวจสอบ input ทุกตัวก่อนประมวลผล คืน 422 พร้อม errors object เมื่อข้อมูลไม่ถูกต้อง

สร้าง Module ใหม่

เพิ่ม API Endpoint ใหม่ใน 2 ไฟล์

# โครงสร้างไฟล์สำหรับ module ใหม่
modules/
└── mymodule/
    ├── controllers/
    │   └── user.php    ← /api/mymodule/user/{action}
    └── models/
        └── user.php    ← ตรรกะและ database queries

─────── modules/mymodule/controllers/user.php ───────
namespace Mymodule\User;

use Kotchasan\Http\Request;

class Controller extends \Kotchasan\ApiController
{
    public function index(Request $request): array
    {
        $users = \Mymodule\User\Model::getAll();
        return ['success' => true, 'data' => $users];
    }

    public function create(Request $request): array
    {
        // ตรวจสอบ JWT middleware ก่อน
        $data = $request->getParsedBody();
        return \Mymodule\User\Model::create($data);
    }
}

─────── modules/mymodule/models/user.php ────────────
namespace Mymodule\User;

use Kotchasan\Model;

class Model extends \Kotchasan\Model
{
    public static function getAll(): array
    {
        return \Kotchasan\DB::table('user')->get();
    }

    public static function create(array $data): array
    {
        \Kotchasan\DB::table('user')->insert($data);
        return ['success' => true];
    }
}

─────── เรียกใช้ ───────
GET  /api/mymodule/user/index   → Controller::index()
POST /api/mymodule/user/create  → Controller::create()
